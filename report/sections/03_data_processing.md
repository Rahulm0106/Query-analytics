# Section 3 - Data Processing

The Query Analytics Workloads dataset ships as three sub files derived from Chicago crime records: Range Queries Aggregates (200,000 rectangular range queries with Count, SUM, and AVG targets), Radius Queries (50,000 disc queries with only geometric parameters), and Radius Queries Count (10,000 disc queries with a Count target). Every row was parsed in R with readr, and observed row counts exactly matched the totals reported on the UCI page (260,000 combined). The full pipeline is reproducible from src/preprocess.R and narrated in notebooks/01_preprocessing.Rmd.

Schema validation surfaced one anomaly the UCI card does not document: 157 rows in Range Queries Aggregates have AVG = NA because Count = 0 and SUM = 0 make the ratio undefined. These are structural zeros rather than data quality failures, so AVG was imputed to 0 for those rows; no other nulls existed in any file. Exact duplicate detection removed zero rows across all three files (100% retention). A boundary effect check flagged queries whose coverage extends past the per dataset bounding box: 1,373 of the 200,000 range queries and effectively all radius queries, a direct consequence of the Gaussian sampling used to synthesize the workloads.

Feature engineering added four spatial features per row: area (4·X_range·Y_range for rectangles; π·R² for discs), per dataset min max normalized X_norm and Y_norm ∈ [0, 1], and dist_centroid, the Euclidean distance from each query center to the dataset mean. Because Count and SUM are heavily right skewed (skew > 3), log1p transforms of both targets were exported alongside the raw variants so Day 3 can model on whichever scale gives the best residuals.
